◆ mdb_on_error_t

typedef void(* mdb_on_error_t) (mdb_error_t *error, void *user_data)

Error callback

When this function is called the associated object is deemed unusable and should be destroyed.

In this callback function, one must not free the error with mdb_error_destroy(), since the library will do it when returning from this callback.

Enumeration Type Documentation

◆ mdb_error_code_t

enum mdb_error_code_t

Enum type for error codes.

These are set in mdb_error_t on errors. Note that only the negative values are defined in this enum explicitly. mdb_error_code_t can also be a positive value, this corresponds then to Linux style errno values.

void mdb_error_destroy ( mdb_error_t **  error)

Deallocate an error handle.

Functions that has an argument of type mdb_error_t** allocates mdb_error_t when errors occur. This function should then later be used to free it.

Parameters
errorAn initialized error handle or NULL. If error is NULL this function will do nothing. *error will be set to NULL after this call.
