Detailed Description

The main include for the AXHttp library.

Introduction

The AXHttp library provides a simple mechanism for serving CGI requests. An ACAP application that wants to utilize the AXHttp library must provide a file containing the paths to each CGI it wants to register and assign the name of that file to the HTTPCGIPATHS variable in the package.conf file.

A CGI configuration file has the following form

<access_level> <path>
<access_level> <path>
<...>

where the access_level must be one of

  • viewer
  • operator
  • administrator

and where path is a valid CGI path name.

Example 

viewer /detectors/list.cgi
operator /detectors/add.cgi
operator /detectors/remove.cgi

When the application is installed the installation script will create the paths listed in the configuration file with the given access level and register them with the HTTP server.

In order to avoid conflicts with other applications, each script will be prefixed by the string /local/<appname>/ where <appname> is the name of the application as set in package.conf. A client that wants to send a request to a registered CGI must take the prefix into account. That is, a request to the script detectors/list.cgi should be sent to "http://<server>/local/<appname>/detectors/list.cgi"

Serving requests

The set of CGI:s an application can register is determined by the configuration file. CGI:s can not be added or removed dynamically.

Before any requests can be served, the HTTP handler must first be set up by calling ax_http_handler_new() which returns an AXHttpHandler reference. Only one call to ax_http_handler_new() is necessary.

The web server will forward all requests to the registered CGI:s to the application, providing

  • The path to the requested CGI as listed in the configuration file.
  • The method used to make the request.
  • The query in raw form.
  • A Hashtable where the query has been parsed into keys and values.
  • An output stream to write resonses to.

The application is responsible for replying to the request in accordance with "The Common Gateway Interface (CGI) Version 1.1" RFC 3875.

Shutting down

When an application no longer wishes to handle requests it should call ax_http_handler_free() an pass in the reference to the AXHttpHandler it previously obtained from ax_http_handler_new().
